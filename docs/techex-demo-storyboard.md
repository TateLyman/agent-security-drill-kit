# TechEx Demo Storyboard

Target length: 90 seconds.

## Opening

Voiceover:

> Agents are starting to touch files, APIs, private data, and payments, but most demos still cannot prove what happens when something goes wrong. Agent Security Drill Kit turns that trust gap into a local evidence pack.

Screen:

1. Open `https://tateprograms.com/agent-security-drill.html`.
2. Show the terminal header and the first call to action.
3. Set project name to `Support Agent Trust Demo`.
4. Paste or load the fixture from `docs/demo-agent-project`.

## Middle

Voiceover:

> The scanner checks the boring controls that matter: policy actions, prompt-injection drills, exfiltration handling, network and filesystem boundaries, human review, rate limits, safe demo mode, audit logs, A2A identity, Gemini key safety, tests, CI, and secret hygiene.

Screen:

1. Show the score card.
2. Scroll through grouped signals.
3. Highlight the policy file, A2A Agent Card, red-team drills, and audit event schema from the fixture.
4. Open the fix queue and show that every result is evidence-backed rather than a generic claim.

## Evidence Export

Voiceover:

> The useful output is not just a score. The export gives a Lobster Trap-style policy starter, A2A-style Agent Card, audit schema, Gemini adversarial-test prompt, fix backlog, and a concise submission summary.

Screen:

1. Click `copy evidence pack`.
2. Click `copy submission summary`.
3. Show the artifact pack section with policy, A2A, audit schema, and Gemini drill prompt.

## Close

Voiceover:

> The buyer story is simple: before an agent team asks a sponsor, enterprise buyer, or security reviewer to trust the workflow, they can show what gets denied, what needs human review, and what audit evidence is preserved.

Screen:

1. End on the project URL and repo URL.
2. Keep the final frame on `https://tateprograms.com/agent-security-drill.html`.
